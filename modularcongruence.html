    Section 5.2 Congruence and Modular Operations

    Now that we have developed the concept of equivalence relations and equivalence classes, we can use this to construct systems of modular arithmetic.
    🔗

    Definition 5.2.1.

    Let \(a,b,n \in \mathbb{Z}\) with \(n \gt 0\text{.}\) We say that \(a\) is congruent to \(b\) modulo \(n\text{,}\) written
    \begin{equation*} a \equiv b \pmod{n}\, , \end{equation*}
    if \(n \mid (a-b)\text{.}\) In other words, \(a\) and \(b\) are congruent modulo \(n\) if their difference is divisible by \(n\text{,}\) which is the same as saying that \(a\) and \(b\) have the same remainder when divided by \(n\text{.}\)
    🔗
    🔗
    For example, we have that \(17 \equiv 5 \pmod{12}\text{,}\) because \(17-5=12\) is divisible by \(12\text{.}\) We also have that \(17 \equiv 29 \pmod{12}\text{,}\) because \(17-29=-12\) is divisible by \(12\text{.}\) Thus, the set of integers that are congruent to \(17\) modulo \(12\) is
    \begin{equation*} \{17+12t: t\in \mathbb{Z}\} = \{\ldots, -7, 5, 17, 29, 41, \ldots\} \, . \end{equation*}
    🔗

    Theorem 5.2.2.

    The relation of congruence modulo \(n\) is an equivalence relation on \(\mathbb{Z}\text{.}\)
    🔗
    🔗

    Proof.

    We need to show that the relation of congruence modulo \(n\) is reflexive, symmetric, and transitive.
    1. To show that the relation is reflexive, we need to show that \(a \equiv a \pmod{n}\) for all \(a \in \mathbb{Z}\text{.}\) This is true because \(a-a=0\) is divisible by \(n\text{.}\)
      🔗
      🔗
    2. To show that the relation is symmetric, we need to show that if \(a \equiv b \pmod{n}\text{,}\) then \(b \equiv a \pmod{n}\text{.}\) This is true because if \(a \equiv b \pmod{n}\text{,}\) then \(n \mid (a-b)\text{,}\) which means that \(n \mid (b-a)\text{,}\) which means that \(b \equiv a \pmod{n}\text{.}\)
      🔗
      🔗
    3. To show that the relation is transitive, we need to show that if \(a \equiv b \pmod{n}\) and \(b \equiv c \pmod{n}\text{,}\) then \(a \equiv c \pmod{n}\text{.}\) Observe that if \(a \equiv b \pmod{n}\text{,}\) then \(n \mid (a-b)\text{.}\) Also, if \(b \equiv c \pmod{n}\text{,}\) then \(n \mid (b-c)\text{.}\) Adding these values, we have \(n \mid ((a-b)+(b-c))\text{,}\) which means that \(n \mid (a-c)\text{.}\) Thus, we have \(a \equiv c \pmod{n}\text{.}\)
      🔗
      🔗
    🔗
    🔗

    We next wish to introduce an arithmetic structure on the equivalence classes of integers modulo \(n\text{.}\)
    🔗

    Definition 5.2.5.

    Let \(n\gt 0\text{.}\) We write the set of equivalence classes of integers modulo \(n\) as \(\mathbb{Z}/n\mathbb{Z}\text{,}\) which we call "Z mod nZ". We define addition and multiplication on \(\mathbb{Z}/n\mathbb{Z}\) as follows. For \([a],[b] \in \mathbb{Z}/n\mathbb{Z}\text{,}\) we define
    \begin{equation*} [a]+[b] := [a+b] \end{equation*}
    and
    \begin{equation*} [a][b] := [ab]\text{.} \end{equation*}
    We call the resulting system of equivalence classes with these operations of addition and multiplication the modular arithmetic system modulo \(n\text{.}\)
    🔗
    🔗

    As with the case of equivalent fractions, we need to prove that the operations of addition and multiplication on equivalence classes of integers modulo \(n\) are well-defined, meaning that the result does not depend on the choice of representatives of the equivalence classes we are adding or multiplying together.
    🔗

    Theorem 5.2.7.

    Modular arithmetic is well-defined.
    🔗
    🔗

    Proof.

    We first need to prove that addition is well-defined modulo \(n\text{.}\) We need to show that if \([a] = [a']\) and \([b] = [b']\text{,}\) then \([a+b] = [a'+b']\text{.}\) Because \([a] = [a']\text{,}\) we have that \(n \mid (a-a')\text{,}\) and similarly for \([b]=[b']\text{.}\) Thus,
    \begin{equation*} n \mid (a-a' + b-b') = ((a+b) - (a'+b'))\, , \end{equation*}
    which means that \([a+b] = [a'+b']\text{.}\)
    🔗
    Next, we need to prove that multiplication is well-defined modulo \(n\text{.}\) We need to show that if \([a] = [a']\) and \([b] = [b']\text{,}\) then \([ab] = [a'b']\text{.}\) Because \([a] = [a']\text{,}\) we have that \(n \mid (a-a')\text{,}\) and similarly for \([b]=[b']\text{.}\) Thus,
    \begin{equation*} n \mid ((a-a')b + a'(b-b')) = ab - a'b + a'b - a'b' = ab-a'b' \, , \end{equation*}
    which means that \([ab] = [a'b']\text{.}\)
    🔗
    🔗

    Checkpoint 5.2.8.

    Discuss the proof above. Does it make sense? Why or why not?
    🔗
    🔗
    Because modular arithmetic is well-defined, we can treat the equivalence classes of integers modulo \(n\) as if they were independent objects, and we can add and multiply them together by selecting representatives of each class and get the same result regardless of which representatives we choose to add or multiply together. In practice, we usually use the values
    \begin{equation*} \{0,1,2,3,\ldots,n-1\}\text{,} \end{equation*}
    which we call distinguished representatives, as the representatives for each equivalence class. When we add these values, we "reduce mod \(n\)", meaning that if we end up with a value larger than \(n\text{,}\) we replace that number with the remainder after dividing by \(n\text{.}\)
    🔗
    A common convention for representing the equivalence classes modulo \(n\) is to write the values \(\{0,1,2,3,\ldots,n-1\}\) on a clock. For example, the equivalence classes of integers modulo \(12\) are represented on a clock with the numbers \(0\) through \(11\text{,}\) and we add by moving clockwise around the clock.

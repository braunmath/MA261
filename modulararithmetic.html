    Section 5.3 Modular Arithmetic

    Because modular operations rely on addition and multiplication in the integers, we can use the properties of addition and multiplication in the integers to deduce properties of addition and multiplication in modular arithmetic. This leads to the following theorem, which we will not prove in detail.
    🔗

    Theorem 5.3.1.

    Modular addition and multiplication are commutative, associative, and the distributive law holds. Further, the cancellation law holds for addition, meaning that if \([a]+[c] = [b]+[c]\text{,}\) then \([a] = [b]\text{.}\) An equivalent way to state the cancellation law is that if
    \begin{equation*} a + c \equiv b +c \pmod{n} \, , \end{equation*}
    then
    \begin{equation*} a \equiv b \pmod{n} \, . \end{equation*}
    🔗
    🔗

    Checkpoint 5.3.2.

    Discuss with your group: why does it make sense that the cancellation law holds?
    🔗
    🔗
    There are some significant differences between arithmetic in the integers and modular arithmetic. The first major difference is that it is possible to multiply two nonzero equivalence classes together and get the zero equivalence class. For example, in \(\mathbb{Z}/6\mathbb{Z}\text{,}\) we have that \([2][3] = [6] = [0]\text{.}\)
    🔗

    Definition 5.3.3.

    A zero divisor in a modular arithmetic system is a nonzero equivalence class \([a]\) such that there exists a nonzero equivalence class \([b]\) with \([a][b] = [0]\text{.}\)
    🔗
    🔗

    Checkpoint 5.3.4.

    Find all of the zero divisors in \(\mathbb{Z}/12\mathbb{Z}\text{.}\)
    🔗
    🔗
    Another major difference is that it is not always possible to divide in modular arithmetic. For example, in \(\mathbb{Z}/6\mathbb{Z}\text{,}\) we have that \([2][3] = [0]\text{,}\) so if we try to divide both sides by \([2]\text{,}\) we get \([3] = [0]\text{,}\) which is not true. This leads to the following definition.
    🔗

    Definition 5.3.5.

    A unit in \(\mathbb{Z}/n\mathbb{Z}\) is an equivalence class \([a]\neq [0]\) such that there exists an equivalence class \([b]\) with \([a][b] = [1]\text{.}\) In this case, we call \([b]\) the multiplicative inverse of \([a]\text{,}\) and denote it as \([a]^{-1}\text{.}\) Equivalently, \([a]\) is a unit if there exists a non-zero \(b\) such that
    \begin{equation*} ab \equiv 1 \pmod{n} \, . \end{equation*}
    🔗
    🔗
    Observe that a zero divisor cannot be a unit, because if \([a]\) is a zero divisor, then there exists a nonzero equivalence class \([b]\) such that \([a][b] = [0]\text{.}\) So, if \([a]\) were a unit, we would have that
    \begin{equation*} [0] = [a]^{-1}[0] = [a]^{-1}[a][b] = [1][b] = [b]\, \end{equation*}
    which contradicts our assumption that \([b]\neq [0]\text{.}\)
    🔗

    Checkpoint 5.3.6.

    Find all of the units in \(\mathbb{Z}/12\mathbb{Z}\) and their multiplicative inverses.
    🔗
    🔗
    From the checkpoint above, you might conjecture the following theorem.
    🔗

    Theorem 5.3.7.

    In \(\mathbb{Z}/n\mathbb{Z}\text{,}\) an equivalence class \([a]\) is a unit if and only if \(\gcd(a,n)=1\text{.}\)
    🔗
    🔗

    Proof.

    We first prove the forward direction. Assume that \([a]\) is a unit in \(\mathbb{Z}/n\mathbb{Z}\text{.}\) Then there exists an equivalence class \([b]\) such that \([a][b] = [1]\text{.}\) This means that \(ab \equiv 1 \pmod{n}\text{,}\) which means that \(n \mid (ab-1)\text{.}\) Thus, there exists an integer \(t\) such that \(ab-1 = nt\text{,}\) which means that \(ab + nt = 1\text{.}\) This is the Bezout identity, which implies that \(\gcd(a,n)=1\text{.}\)
    🔗
    We next prove the reverse direction. Assume that \(\gcd(a,n)=1\text{.}\) Then there exist integers \(x\) and \(y\) such that \(ax + ny = 1\text{.}\) This means that \(ax \equiv 1 \pmod{n}\text{,}\) which means that \([a][x] = [1]\text{.}\) Thus, \([a]\) is a unit in \(\mathbb{Z}/n\mathbb{Z}\text{.}\)
    🔗
    🔗

    Checkpoint 5.3.8.

    Discuss the proof above. Does it make sense? Why or why not?
    🔗
    🔗
    When \(n\) is a prime number, the condition \(\gcd(a,n)=1\) is always true for \(a=1,2,\ldots,n-1\text{.}\) In this case, every non-zero equivalence class is a unit. We record this formally below.
    🔗

    Corollary 5.3.9.

    If \(p\) is prime, then every non-zero equivalence class in \(\mathbb{Z}/p\mathbb{Z}\) is a unit.
    🔗
    🔗

    Proof.

    This follows immediately from the fact that if \(p\) is prime, then \(\gcd(a,p)=1\) for all \(a=1,2,\ldots,p-1\text{.}\)
    🔗
    🔗
    So, in \(\mathbb{Z}/p\mathbb{Z}\text{,}\) we can add, subtract, multiply, and divide (except by \([0]\)) without every causing any issues. This is one of the reasons why working modulo a prime is such a powerful tool in mathematics. (In modern algebra classes like MA 361-362, you learn that \(\mathbb{Z}/p\mathbb{Z}\) is a field, while in general the modular arithmetic systems \(\mathbb{Z}/n\mathbb{Z}\) are rings but not fields.)
    🔗
    We use the Euclidean algorithm and the Bezout identity to find the multiplicative inverse of an equivalence class \([a]\) in \(\mathbb{Z}/n\mathbb{Z}\) when \(\gcd(a,n)=1\text{.}\) Specifically, we use the Euclidean algorithm to find integers \(x\) and \(y\) such that \(ax + ny = 1\text{.}\) Then we have that \(ax \equiv 1 \pmod{n}\text{,}\) which means that \([a][x] = [1]\text{,}\) so \([x]\) is the multiplicative inverse of \([a]\text{.}\)
    🔗
    For example, take \([7]\) in \(\mathbb{Z}/13\mathbb{Z}\text{.}\) Since \(\gcd(7,13)=1\text{,}\) we know that \([7]\) is a unit in \(\mathbb{Z}/13\mathbb{Z}\text{.}\) We use the Euclidean algorithm to find integers \(x\) and \(y\) such that \(7x + 13y = 1\text{.}\) We have:
    \begin{gather*} 13 = 7\cdot 1 + 6\\ 7 = 6\cdot 1 + 1\\ 6 = 1\cdot 6 + 0 \end{gather*}
    Thus, we have that
    \begin{equation*} 1 = 7 - 6\cdot 1 = 7 - (13 - 7\cdot 1) = 7\cdot 2 - 13\cdot 1 \, . \end{equation*}
    This means that \(7\cdot 2 \equiv 1 \pmod{13}\text{,}\) so \([7][2] = [1]\text{,}\) and thus \([2]\) is the multiplicative inverse of \([7]\) in \(\mathbb{Z}/13\mathbb{Z}\text{.}\)
    🔗

    Checkpoint 5.3.10.

    Find the multiplicative inverse of \([30]\) in \(\mathbb{Z}/107\mathbb{Z}\) using the Euclidean algorithm and the Bezout identity.
    🔗
    🔗
    A key role played by units is that these are the only equivalence classes that we can cancel in modular arithmetic.
    🔗

    Theorem 5.3.11.

    Suppose that \([a]\) is a unit in \(\mathbb{Z}/n\mathbb{Z}\text{,}\) and suppose that \([a][b] = [a][c]\text{.}\) Then \([b] = [c]\text{.}\)
    🔗
    🔗

    Proof.

    Because \([a]\) is a unit, there exists an equivalence class \([d]\) such that \([a][d] = [1]\text{.}\) Thus if \([a][b] = [a][c]\text{,}\) we have that
    \begin{equation*} [b] = [1][b] = [a][d][b] = [d][a][b] = [d][a][c] = [a][d][c] = [1][c] = [c]\, . \end{equation*}
    🔗
    🔗

    Checkpoint 5.3.12.

    Discuss the proof above. Does it make sense? Why or why not?
    🔗
    🔗
    Observe another interesting property of modular arithmetic: if we add a nonzero equivalence class to itself enough times, we can get the zero equivalence class. For example, in \(\mathbb{Z}/6\mathbb{Z}\text{,}\) we have that \([2]+[2]+[2] = [6] = [0]\text{.}\) In general, we have that in \(\mathbb{Z}/n\mathbb{Z}\text{,}\) if we add \([a]\) to itself \(n\) times, we get \([0]\text{.}\) This leads to the definition of the additive order of an equivalence class in modular arithmetic.
    🔗

    Definition 5.3.13.

    Let \([a] \in \mathbb{Z}/n\mathbb{Z}\text{.}\) The additive order of \([a]\) is the smallest positive integer \(k\) such that
    \begin{equation*} [a]+[a]+\cdots+[a] = [0]\, , \end{equation*}
    where there are \(k\) copies of \([a]\) in the sum. We denote the additive order by \(|[a]|\text{.}\)
    🔗
    🔗

    Checkpoint 5.3.14.

    Find the additive order of each equivalence class in \(\mathbb{Z}/12\mathbb{Z}\text{.}\)
    🔗
    🔗
    From the examples you just worked out, we can see the following interesting fact: the order of an equivalence class in \(\mathbb{Z}/n\mathbb{Z}\) divides \(n\text{.}\)
    🔗

    Theorem 5.3.15.

    Let \([a] \in \mathbb{Z}/n\mathbb{Z}\text{.}\) Then \(|[a]|=n/\gcd(a,n)\text{,}\) and thus the additive order divides \(n\text{.}\)
    🔗
    🔗

    Proof.

    We set \(d=\gcd(a,n)\text{.}\) Then we can write \(a = da'\) and \(n = dn'\) for some integers \(a'\) and \(n'\) with \(\gcd(a',n')=1\text{.}\) We need to show that the smallest positive integer \(k\) such that
    \begin{equation*} [a]+[a]+\cdots+[a] = [0]\, , \end{equation*}
    where there are \(k\) copies of \([a]\) in the sum, is equal to \(n/\gcd(a,n) = n/d = n'\text{.}\) Observe that
    \begin{equation*} [a]+[a]+\cdots+[a] = [ka] = [kda']\, . \end{equation*}
    If we want this to be equal to \([0]\text{,}\) we need \(n\mid ka\text{,}\) which is the same as \(dn' \mid kda'\text{,}\) which is the same as \(n' \mid ka'\text{.}\) Because \(\gcd(a',n')=1\text{,}\) we have that \(n' \mid ka'\) if and only if \(n'\) divides \(k\text{.}\) Thus, the smallest positive integer \(k\) such that \(n'\mid ka'\) is \(n'\text{,}\) which means that the smallest positive integer \(k\) such that
    \begin{equation*} [a]+[a]+\cdots+[a] = [0]\, , \end{equation*}
    where there are \(k\) copies of \([a]\) in the sum, is \(n'\text{.}\) Thus, the additive order of \([a]\) is \(n/d = n/\gcd(a,n)\text{,}\) which divides \(n\text{.}\)
    🔗
    🔗

    Checkpoint 5.3.16.

    Discuss the proof above. Does it make sense? Why or why not?
